## 演讲稿（配合上述 PPT 页面）

**[开场过渡]**
接下来，我想重点介绍一下我们在鲲鹏平台上的深度性能优化工作。我们没有简单地依赖硬件堆砌，而是选择深入挖掘鲲鹏 920 处理器的通用计算潜能。

**[第一点：全链路 ONNX 与指令集融合]**
首先，在核心的 ASR 推理环节，我们实施了**全链路 ONNX 化**改造。
我们摒弃了厚重的深度学习框架，直接采用轻量级的 **ONNX Runtime** 作为推理引擎。这一选择在鲲鹏平台上产生了奇妙的化学反应：ONNX Runtime 的底层计算库（MLAS）能够与鲲鹏 CPU **自动握手**，直接激活 **ARM NEON** 向量指令集。
这意味着，每一次语音识别请求，都在以硬件级优化的汇编指令高效运行。我们的实测数据显示，相比传统方案，推理延迟降低了 **50% 以上**。这不仅是软件架构的胜利，更是对鲲鹏通用算力潜能的极致挖掘。

**[第二点：多核亲和性]**
其次，针对鲲鹏处理器众核、多 NUMA 节点的架构特点，我们专门设计了**多核亲和性调度**方案。
在实时语音交互中，线程的频繁切换是性能杀手。我们通过软件配置，将推理任务“锁死”在特定的物理核心上（Pinning），并确保内存分配在本地 NUMA 节点。
这样做直接消除了跨节点访问内存的延迟，也避免了操作系统调度带来的抖动，让每一次语音识别的耗时都极其稳定。

**[第三点：总结]**
最后，这种 `cpu-onnx` 的方案体现了极高的工程价值。它不需要依赖特定的 NPU 固件或复杂的驱动链，完全基于 openEuler 和鲲鹏通用算力，实现了“一次构建，随处部署”。
这不仅降低了系统的运维成本，也完美契合了本次大赛“端云协同”的技术理念——云端训练模型，通过 ONNX 标准格式，无缝下发到鲲鹏边缘侧高效运行。
